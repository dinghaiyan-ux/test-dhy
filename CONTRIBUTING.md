# Contributing to test-dhy

Thank you for considering contributing to this project!

## Workflow

1. **Branch** — Create a feature branch from `main` using the naming convention `feature/<description>`.
2. **Commit** — Write clear, concise commit messages describing your changes.
3. **Pull Request** — Open a PR targeting `main`. Include a summary of changes in the description.
4. **Review** — All PRs require at least one review before merging.

## Code Standards

- Keep documentation clear and up to date
- Follow consistent formatting across all files
- Test changes before submitting a PR

## Reporting Issues

Open an issue in this repository with a clear description of the problem and steps to reproduce.
